# دليل نظام إدارة الطلاب (Spring Boot CRUD)


---

## 🔗 الروابط والعمليات (Endpoints & Operations)

### 1. الصفحة الرئيسية (عرض الطلاب وفورم الإضافة)
* **الرابط:** `GET /students`
* **العملية (CRUD):** Read (قراءة البيانات)
* **كيف تم برمجتها:** * استخدمنا `@GetMapping` لالتقاط طلبات الدخول العادية عبر المتصفح.
    * تقوم الدالة باستدعاء `studentRepository.findAll()` لجلب قائمة بجميع الطلاب من قاعدة البيانات.
* **كيف تعمل:**
    * يتم استخدام `StringBuilder` لبناء صفحة HTML كـ String.
    * يتم رسم "نموذج" (Form) يحتوي على حقول الاسم، التخصص، والدرجة، ويشير إلى رابط الإضافة `POST /students/add`.
    * يتم عمل حلقة تكرار (`for-loop`) للمرور على قائمة الطلاب، ويتم إنشاء صف (`<tr>`) في جدول HTML لكل طالب لعرض بياناته مع زر لحذفه.

---

### 2. إضافة طالب جديد
* **الرابط:** `POST /students/add`
* **العملية (CRUD):** Create (إنشاء/إضافة)
* **كيف تم برمجتها:**
    * استخدمنا `@PostMapping` لأن البيانات قادمة من نموذج HTML (Form) ولحماية البيانات.
    * استخدمنا `@RequestParam` لاستلام البيانات من الحقول (`name`, `major`, `grade`).
* **كيف تعمل:**
    * عندما يضغط المستخدم على زر "Add Student"، يتم إرسال البيانات إلى هذا الرابط.
    * يستقبل الـ Controller المتغيرات ويمررها إلى الدالة `studentRepository.add(...)`.
    * داخل الـ Repository، يتم إنشاء كائن `Student` جديد وحفظه باستخدام `em.persist()`.
    * أخيراً، يُرجع الـ Controller صفحة HTML بسيطة تخبر المستخدم بنجاح العملية مع رابط للعودة للصفحة الرئيسية.

---

### 3. حذف طالب
* **الرابط:** `GET /students/delete/{id}`
* **العملية (CRUD):** Delete (حذف)
* **كيف تم برمجتها:**
    * استخدمنا `@GetMapping` لسهولة استدعاء الرابط مباشرة من جدول HTML عبر وسم `<a>`.
    * استخدمنا `@PathVariable` لاستخراج رقم الطالب (ID) من الرابط نفسه.
* **كيف تعمل:**
    * بمجرد الضغط على زر "Delete" بجانب اسم الطالب، يتم توجيه المتصفح إلى رابط مثل `/students/delete/5`.
    * يأخذ الـ Controller الرقم `5` ويمرره إلى الدالة `studentRepository.delete(5)`.
    * يبحث الـ Repository عن الكائن بواسطة الـ ID، وإذا وجده يقوم بحذفه باستخدام `em.remove()`.
    * يتم إرجاع رسالة تأكيد أو خطأ بناءً على نتيجة العملية.

---

### 4. تحديث درجة الطالب (مثال إضافي)
* **الرابط:** `GET /students/update-grade/{id}?newGrade=X`
* **العملية (CRUD):** Update (تعديل)
* **كيف تم برمجتها:**
    * يجمع هذا الرابط بين `@PathVariable` لجلب (ID)، و `@RequestParam` لجلب القيمة الجديدة للدرجة (`newGrade`).
* **كيف تعمل:**
    * يقوم الـ Controller أولاً بجلب الطالب عبر `studentRepository.findById()`.
    * إذا كان الطالب موجوداً، يقوم بتغيير درجته عبر `student.setGrade(newGrade)`.
    * ثم يمرر الكائن المعدل إلى `studentRepository.update()` الذي يستخدم `em.find` لجلب الكائن المدار (Managed Entity) وتحديث حقوله ليقوم JPA بحفظ التعديل تلقائياً في قاعدة البيانات.

## ⚙️ كيف تتواصل المكونات معاً (Architecture)

يتكون النظام من ثلاث طبقات رئيسية تتفاعل مع بعضها:

1.  **الكيان (Entity - `Student.java`):**
    * يمثل الهيكل الأساسي للبيانات (الجدول في قاعدة البيانات).
    * يستخدم أنوتيشن `@Entity` ليفهم Spring أنه جدول، و `@Id` لتحديد المفتاح الأساسي (Primary Key).

2.  **المستودع (Repository - `StudentRepository.java`):**
    * يحتوي على الأوامر المباشرة للتعامل مع قاعدة البيانات باستخدام `EntityManager`.
    * يحمل أنوتيشن `@Transactional`، وهي مسؤولة عن إدارة الـ (Transactions) تلقائياً، فلا داعي لكتابة `em.getTransaction().begin()` و `commit()` كما في تطبيقات Java العادية.

3.  **المتحكم (Controller - `StudentController.java`):**
    * الواجهة التي تتلقى طلبات المتصفح.
    * يقوم بحقن المستودع بداخله باستخدام `@Autowired` ليتمكن من استدعاء دوال الإضافة والحذف والقراءة.
    * يحمل أنوتيشن `@RestController` الذي يقوم بتحويل أي نص يُرجعه (مثل كود HTML) مباشرة إلى المتصفح دون الحاجة لمحركات قوالب مثل Thymeleaf.
